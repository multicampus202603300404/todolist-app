### Vercel 배포 — Swagger UI 미표시 문제

> 💬 **[프롬프트]**
> "스웨거 ui 가 안보여 어떻하지" (Vercel 배포 후)

**원인 분석**:
1. `vercel.json` 없음 → 모든 요청이 Express로 라우팅되지 않음
2. Swagger UI 정적 파일이 서버리스 환경에서 제대로 서빙되지 않음

**해결**:
- `backend/vercel.json` 생성: 모든 요청을 `src/server.js`로 라우팅
- Swagger UI를 CDN 방식으로 변경 (`unpkg.com/swagger-ui-dist@5`)

```json
// backend/vercel.json
{
  "version": 2,
  "builds": [{ "src": "src/server.js", "use": "@vercel/node" }],
  "routes": [{ "src": "/(.*)", "dest": "src/server.js" }]
}
```

### Swagger UI 서버 URL 문제

> 💬 **[프롬프트]**
> "스웨거 ui 의 대상 아이피는 버셀의 아이피로 해야 하지 않아? 아니면 선택할 수 있는 아이피가 로컬과 버셀 두가지가 나오던지"

**문제**: Swagger UI의 Servers 드롭다운이 `http://localhost:4000`만 표시됨

> 💬 **[프롬프트]**
> "환경변수로 처리해야 하지 않아"

**해결**: `app.js`에서 `VERCEL_URL` 환경변수(Vercel이 자동 제공)를 활용하여 서버 목록을 동적 구성

```javascript
// 환경변수 기반 동적 서버 목록
const swaggerServers = [];
if (process.env.VERCEL_URL) {
  swaggerServers.push({ url: `https://${process.env.VERCEL_URL}`, description: 'Production server (Vercel)' });
}
swaggerServers.push({ url: 'http://localhost:4000', description: 'Local development server' });
```

- Vercel 환경: Production + Local 두 가지 선택 가능
- 로컬 환경: Local만 표시

## 핵심 학습 포인트

### 1. 에이전트 기반 개발 (Agent-Driven Development)
- 전문 서브에이전트에 역할을 분담하여 **병렬 작업** 수행
- 한 번의 프롬프트로 여러 에이전트를 동시에 호출하여 BE/FE/DB 병렬 구현
- 에이전트가 코드를 직접 생성하고, 테스트를 작성하고, 빌드를 검증하는 **자율적 개발 사이클** 수행

### 2. 문서 우선 개발 (Document-First Development)
- 코드 작성 전에 **9종의 설계 문서**를 먼저 완성
- 문서 품질을 **작성→평가→개선→재평가** 사이클로 반복 향상 (8.4/10 → 9.2/10)
- 구현 단계에서 문서를 참조하여 **일관된 코드 품질** 확보

### 3. 교차 검토 (Cross-Validation)
- 문서 간 **일관성 검증**이 구현 단계의 오류를 사전 방지
- PRD ↔ 시나리오 ↔ swagger ↔ 코드 4방향 검토로 **12건 이상의 불일치** 발견 및 수정

### 4. MCP 활용
- `postgresql-mcp`로 DB를 **Claude Code 내에서 직접 조작**
- `mcp__supabase__*`로 클라우드 DB 마이그레이션 자동화
- 마이그레이션 실행 결과를 즉시 검증하는 **하나의 대화 흐름**에서 완료

### 5. 실행계획 기반 추적 (Plan-Driven Tracking)
- 45개 Task를 **의존성 그래프**로 정렬하여 병렬 실행 순서 결정
- 268개 **체크박스**로 진행률 가시화 및 완료 기준 명확화

### 6. 프롬프트 엔지니어링 실전 패턴
- **역할 지정**: "너는 최고의 비즈니스 도메인 전문가야"
- **서브에이전트 위임**: "적절한 서브에이전트를 활용해 병렬로 수행해줘"
- **산출물 지정**: "docs/1-domain-definition.md 파일로 저장해줘"
- **평가 기준 제시**: "다음 5가지 항목을 기준으로 평가해줘"
- **점진적 개선**: "우선 개선 권고 사항을 적용하여 재작성해줘"
- **교차 참조**: "@docs/ 디렉토리의 문서와 비교분석하여 일관성을 검토하고 수정해줘"

### 7. 배포 트러블슈팅
- **Mixed Content**: HTTPS 프론트엔드에서 HTTP 백엔드 호출 차단 → `VITE_API_BASE_URL`을 `https://`로 변경
- **Swagger UI 미표시**: 서버리스 환경 정적 파일 서빙 이슈 → CDN 방식으로 전환 + `vercel.json` 추가
