### Vercel 배포 — Swagger UI 미표시 문제 해결

> 💬 **[프롬프트]**  
> "스웨거 ui 가 안보여 어떻하지" (Vercel 배포 후)

**원인**:
1. `vercel.json` 없음 → 모든 요청이 Express로 라우팅되지 않음
2. Swagger UI 정적 파일이 서버리스 환경에서 서빙되지 않음

**해결**:
```json
// backend/vercel.json 신규 생성
{
  "version": 2,
  "builds": [{ "src": "src/server.js", "use": "@vercel/node" }],
  "routes": [{ "src": "/(.*)", "dest": "src/server.js" }]
}
```
Swagger UI 정적 에셋을 **unpkg CDN** 방식으로 변경
```javascript
swaggerUi.setup(swaggerDocument, {
  customCssUrl: 'https://unpkg.com/swagger-ui-dist@5/swagger-ui.css',
  customJs: ['https://unpkg.com/swagger-ui-dist@5/swagger-ui-bundle.js', ...]
})
```

### Vercel 배포 — Swagger UI 서버 URL 문제

> 💬 **[프롬프트]**  
> "스웨거 ui 의 대상 아이피는 버셀의 아이피로 해야 하지 않아? 아니면 선택할 수 있는 아이피가 로컬과 버셀 두가지가 나오던지"

**문제**: Swagger UI Servers 드롭다운이 `http://localhost:4000`만 표시됨

> 💬 **[프롬프트]**  
> "환경변수로 처리해야 하지 않아"

**1차 해결**: `app.js`에서 `VERCEL_URL` 환경변수(Vercel 자동 제공)로 동적 구성  
→ 그러나 `VERCEL_URL`은 배포별 자동 생성 URL(`backend-bhcl6bru6-xxx.vercel.app`)로 표시되는 문제 남음

> 💬 **[프롬프트]**  
> "스웨버.json 을 수정하면 된다는데.."

**2차 해결**: `swagger.json`의 `servers`를 직접 수정하고, `app.js`의 동적 덮어쓰기 코드를 제거

```json
// swagger/swagger.json
"servers": [
  { "url": "http://localhost:4000", "description": "Local development server" },
  { "url": "https://backend-kdk.vercel.app", "description": "vercel backend server" }
]
```

→ **swagger.json 직접 관리 방식**으로 최종 결정  
→ `app.js`는 `swaggerDocument`를 그대로 사용 (런타임 덮어쓰기 없음)

### Vercel 배포 — Mixed Content + CORS 오류

> 💬 **[프롬프트]**  
> "frontend 페이지에서 Access to XMLHttpRequest ... has been blocked by CORS policy ... 'Access-Control-Allow-Origin' header has a value 'http://frontend-kdk.vercel.app:5173'"

**에러 원인 분석**:

| 항목 | 잘못된 값 | 올바른 값 |
|------|----------|----------|
| `VITE_API_BASE_URL` | `http://localhost:4000` | `https://backend-kdk.vercel.app` |
| `CORS_ORIGIN` (백엔드 Vercel 환경변수) | `http://frontend-kdk.vercel.app:5173` | `https://frontend-kdk.vercel.app` |

**Mixed Content 오류**: HTTPS 프론트에서 HTTP 백엔드 호출 → 브라우저 차단  
**CORS 오류**: HTTP + 포트 포함된 Origin이 실제 HTTPS Origin과 불일치

**해결 방법**:
- 프론트엔드 Vercel 프로젝트: `VITE_API_BASE_URL = https://backend-kdk.vercel.app` 설정 후 Redeploy
- 백엔드 Vercel 프로젝트: `CORS_ORIGIN = https://frontend-kdk.vercel.app` 설정 후 Redeploy

| Swagger UI 미표시 | vercel.json 없음 + 서버리스 정적 파일 서빙 불가 | vercel.json 추가 + CDN 방식 전환 |
| Swagger 서버 URL 오표시 | VERCEL_URL이 배포별 임시 URL | swagger.json에 직접 고정 URL 명시 |
| Mixed Content | 프론트(HTTPS)에서 백엔드(HTTP) 호출 | `VITE_API_BASE_URL`을 `https://`로 변경 |
| CORS 오류 | `CORS_ORIGIN`에 HTTP + 포트 포함 | `https://` + 포트 제거로 수정 |

### 8. CI/CD 파이프라인 설계
- Vercel git 자동 배포 비활성화 후 **GitHub Actions로 제어권 이전**
- `backend/**` 경로 변경 시에만 배포 트리거 → **불필요한 배포 방지**
- Secrets(`VERCEL_TOKEN`, `VERCEL_ORG_ID`, `VERCEL_PROJECT_ID_BACKEND`)로 보안 관리
