### 프론트엔드 배포 — SPA 라우팅 404 문제 해결

> 💬 **[프롬프트]**  
> "frontend-app 을 vercel 에 배포 하고 있는 데 fallback ui 를 지정하려고 해, 어떻게 해야할까?"

**SPA 라우팅 문제 설명**:  
React Router를 사용하는 SPA에서 `/todos`, `/profile` 등을 브라우저에서 직접 입력하거나 새로고침하면  
Vercel이 해당 경로의 정적 파일을 찾지 못해 **404 반환**

**해결 방법 3가지 제시**:
1. **Vercel SPA 라우팅 fallback** — `vercel.json`에 rewrites 추가 ✅ 선택
2. React Suspense fallback — 코드 스플리팅 로딩 중 UI
3. React Error Boundary fallback — 런타임 에러 발생 시 UI

> 💬 **[프롬프트]**  
> "커밋 푸쉬 해줘"

**1차 시도** — `vercel.json`에 rewrites 추가:
```json
"rewrites": [{ "source": "/(.*)", "destination": "/index.html" }]
```

> 💬 **[프롬프트]**  
> "여전히 새로고침 해도 404가 나와"

**원인**: `outputDirectory`가 지정되지 않아 Vercel이 Vite 빌드 결과물(`dist/`) 위치를 인식 못함

**최종 해결** — 빌드 설정 추가:
```json
{
  "git": { "deploymentEnabled": false },
  "buildCommand": "npm run build",
  "outputDirectory": "dist",
  "rewrites": [{ "source": "/(.*)", "destination": "/index.html" }]
}
```

- 한 번의 프롬프트로 BE/FE/DB를 동시에 구현하는 **자율적 개발 사이클**

### 2. 문서 우선 개발 (Document-First Development)
- 코드 작성 전 **9종의 설계 문서** 완성
- **작성→평가→개선→재평가** 사이클 (8.4/10 → 9.2/10)

### 3. 교차 검토 (Cross-Validation)
- PRD ↔ 시나리오 ↔ swagger ↔ 코드 **4방향 검토**로 12건+ 불일치 사전 방지

### 4. MCP 활용
- `postgresql-mcp`로 DB를 **Claude Code 내에서 직접 조작** 및 즉시 검증
- `mcp__supabase__*`로 클라우드 DB **마이그레이션 자동화**

### 5. CI/CD 파이프라인 설계
- Vercel git 자동 배포 비활성화 → **GitHub Actions로 제어권 이전**
- `backend/**` / `frontend/**` 경로별 개별 배포 트리거로 **불필요한 배포 방지**
- GitHub Secrets로 Vercel 토큰/프로젝트 ID 보안 관리

### 6. 프롬프트 엔지니어링 실전 패턴

| 패턴 | 예시 |
|------|------|
| 역할 지정 | "너는 최고의 비즈니스 도메인 전문가야" |
| 서브에이전트 위임 | "적절한 서브에이전트를 활용해 병렬로 수행해줘" |
| 산출물 지정 | "docs/1-domain-definition.md 파일로 저장해줘" |

| 완료 조건 강조 | "완료되었다면 검수리스트 체크박스에 체크해야해" |
| 교차 참조 | "@docs/ 디렉토리의 문서와 비교분석하여 일관성을 검토해줘" |

### 7. 배포 환경 이해
- **로컬** vs **서버리스(Vercel)** 환경의 차이점 체득
  - 정적 파일 서빙 방식 차이 → CDN 전환
  - HTTP/HTTPS Mixed Content 정책 이해
  - SPA 라우팅과 서버 설정의 관계 (`outputDirectory` + `rewrites`)
- **환경변수**를 코드에 하드코딩하지 않고 Vercel 대시보드에서 관리
