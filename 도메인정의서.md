# 📑 도메인 정의서: AI 기반 개발 관리자 전용 TODO 시스템

## 1. 핵심 항목 (Core Elements)

### 가. 비즈니스 컨텍스트 (Business Context)
* **목적과 비전**: 개발 실무(Hands-on)와 팀 관리(Management) 업무가 혼재된 환경에서, 프로젝트 진척도를 극대화할 수 있는 최적의 태스크 실행 경로를 제시함.
* **범위 및 경계 설정**: 프로젝트 태스크 관리, 팀 단위 블로킹 이슈 해결, 실무 개발 작업 할당에 집중 (인사/재무 등 경영 영역 제외).
* **이해관계자 매핑**: 플레잉 코치(사용자), 개발 팀원(이슈 발생 주체), 프로젝트 이해관계자(일정 공유 대상).

### 나. 도메인 언어 (Ubiquitous Language)
* **Mission(미션)**: 프로젝트의 성패를 결정짓는 핵심 상위 목표.
* **Blocker(차단 요소)**: 팀원의 업무를 중단시켜 프로젝트 속도를 저하시키는 급박한 기술/관리적 문제.
* **Hands-on(직접 수행)**: 관리자가 직접 코드를 작성하거나 기술 설계를 해야 하는 실무 영역.
* **Backlog(백로그)**: 아직 실행 단계에 진입하지 않은 아이디어 및 잠재적 할 일 더미.

### 다. 도메인 모델 (Domain Model)
* **엔티티(Entity)**: `Task`(할 일), `Project`(프로젝트 단위), `Issue`(발생한 문제).
* **값 객체(Value Object)**: `Impact`(영향도), `Urgency`(긴급도), `TechnicalDebt`(기술 부채 수준).
* **애그리게이트(Aggregate)**: `Project Root` (프로젝트 하위의 모든 태스크와 이슈를 하나의 관리 단위로 묶음).

### 라. 비즈니스 규칙 (Business Rules)
* **불변 규칙**: 모든 실무(Hands-on) 태스크는 반드시 연결된 상위 프로젝트가 존재해야 함.
* **정책**: 'Blocker' 이슈 발생 시 현재 진행 중인 모든 개인 실무 태스크보다 우선순위를 상향 조정함.
* **제약사항**: 관리 업무와 실무 개발 시간의 비율을 설정된 임계치(예: 4:6)에 맞춰 자동 배분함.

### 마. 업무 프로세스 (Business Process)
1. **동기화**: 외부 도구(Jira/GitHub)에서 이슈 및 태스크 데이터 유입.
2. **분석**: 미션별 중요도 및 Blocker 여부에 따른 우선순위 자동 산정.
3. **확정**: 관리자가 AI가 제안한 실행 순서를 검토 및 확정.
4. **수행**: 태스크 수행 및 실시간 진척도 업데이트.
5. **피드백**: 완료된 데이터 기반 프로젝트 대시보드 자동 반영.

## 2. 권장 항목 (Recommended Elements)

### 가. 도메인 이벤트 (Domain Events)
* **Critical Issue Detected**: 프로젝트 진행을 방해하는 중대 결함이나 블로커 발생 시.
* **Task Overdue**: 설정된 마감 기한 내에 실무 태스크가 완료되지 않았을 때.

### 나. 경계 컨텍스트 (Bounded Context)
* **Execution Context**: 실제 태스크의 생성 및 상태 변화 관리.
* **Strategy Context**: 프로젝트 로드맵 기반의 우선순위 전략 수립.
* **External Sync Context**: 외부 협업 도구와의 데이터 정합성 유지.

### 다. 데이터 모델 (Data Model)
* **개념적 구조**: `Project(1) --- (N) Task`, `Task(1) --- (1) PriorityModel`.
* **데이터 카탈로그**: `TaskID`, `Title`, `Type(Management/Dev)`, `Status`, `Effort(예상 공수)`.

### 라. 비기능 요구사항 (Non-Functional Requirements)
* **성능**: 수천 개의 백로그 항목 중 우선순위 재계산 시 1초 이내 처리.
* **가용성**: 오프라인 상태에서도 로컬 캐시를 통한 태스크 조회 지원.
* **확장성**: 다양한 외부 개발 도구(GitLab, Bitbucket 등) API 연동이 용이한 플러그인 구조.

### 마. 예외 처리 (Exception Handling)
* **에지 케이스**: 동일 우선순위 태스크 충돌 시 '마감 임박순'으로 자동 정렬.
* **장애 대응**: 외부 도구 연동 실패 시 '로컬 모드'로 자동 전환하여 데이터 유실 방지.
