## 1. 프로젝트 개요

### 목적

`todolist-app`은 인증된 개인 사용자가 자신만의 할일을 체계적으로 등록, 관리, 추적할 수 있는 개인 맞춤형 할일 관리 애플리케이션이다.

기존 할일 관리 도구의 두 가지 핵심 한계를 해결한다.

- 사용자 간 데이터가 분리되지 않아 개인화된 관리가 불가능한 문제
- 할일의 종료일(마감일)을 명시적으로 지정할 수 없어 일정 추적이 불분명한 문제

### 핵심 가치

| 가치 | 설명 |
|------|------|
| 개인화 | 회원 인증을 기반으로 나만의 할일 공간을 제공한다 |
| 시간 기반 관리 | 시작일과 종료일을 필수로 지정하여 모든 할일에 명확한 기간을 부여한다 |
| 현황 가시성 | 상태 필터링과 통계를 통해 할일의 진행 현황을 한눈에 파악한다 |

## 2. 도메인 모델

### 핵심 엔티티

#### User (사용자)

사용자 인증 및 식별의 기본 단위이다. 모든 할일 데이터는 특정 사용자에게 귀속된다.

| 속성 | 타입 | 설명 | 제약 |
|------|------|------|------|
| id | UUID | 사용자 고유 식별자 | 필수, 자동 생성 |
| email | String | 로그인 이메일 주소 | 필수, 고유값, RFC 5321 이메일 형식 |
| password | String | 암호화된 비밀번호 | 필수, bcrypt 해시 저장, 평문 저장 금지 |
| created_at | DateTime | 가입 일시 | 필수, 자동 생성 |

#### Todo (할일)

사용자가 수행할 개별 작업 단위이다. 반드시 기간(시작일~종료일)을 가진다.

| 속성 | 타입 | 설명 | 제약 |
|------|------|------|------|
| id | UUID | 할일 고유 식별자 | 필수, 자동 생성 |
| user_id | UUID | 소유 사용자 참조 | 필수, User.id 외래키, 수정 불가 |
| title | String | 할일 제목 | 필수, 최대 200자 |
| description | String | 할일 상세 설명 | 선택, 최대 2000자, plaintext |
| start_date | Date | 할일 시작일 | 필수 |
| end_date | Date | 할일 종료일(마감일) | 필수 |
| is_completed | Boolean | 완료 여부 | 필수, 기본값: false |
| completed_at | DateTime | 실제 완료 처리 일시 | 선택, 완료 시 자동 기록, 수정 불가 |
| created_at | DateTime | 등록 일시 | 필수, 자동 생성, 수정 불가 |
| updated_at | DateTime | 최종 수정 일시 | 필수, 자동 갱신 |

> **수정 가능 속성**: title, description, start_date, end_date, is_completed
> **수정 불가 속성**: id, user_id, completed_at, created_at

### 엔티티 관계

```
User (1) ----< Todo (N)
```

- 한 명의 사용자는 0개 이상의 할일을 보유할 수 있다.
- 하나의 할일은 반드시 한 명의 사용자에게만 귀속된다.
- 사용자가 삭제되면 해당 사용자의 모든 할일도 함께 삭제된다 (CASCADE DELETE).

## 4. 할일 상태 정의

할일의 상태는 현재 날짜(TODAY)와 `start_date`, `end_date`, `is_completed` 값을 조합하여 실시간으로 결정된다.

### 상태 종류 및 판별 조건

| 상태 | 상태 코드 | 판별 조건 |
|------|-----------|----------|
| 시작 전 | `UPCOMING` | `is_completed = false` AND `TODAY < start_date` |
| 진행 중 | `IN_PROGRESS` | `is_completed = false` AND `start_date <= TODAY <= end_date` |
| 종료됨 (미완료) | `OVERDUE` | `is_completed = false` AND `TODAY > end_date` |
| 성공적으로 완료 | `COMPLETED_ON_TIME` | `is_completed = true` AND `completed_at <= end_date` |
| 완료하지 못함 | `COMPLETED_LATE` | `is_completed = true` AND `completed_at > end_date` |

### 상태 전환 흐름

```
[UPCOMING] -- start_date 도달 --> [IN_PROGRESS] -- end_date 경과 --> [OVERDUE]
     |                                   |                                |
     | (완료 처리)                        | (완료 처리)                     | (완료 처리)
     v                                   v                                v
[COMPLETED_ON_TIME]            [COMPLETED_ON_TIME]              [COMPLETED_LATE]
                                또는 [COMPLETED_LATE]
```

- 완료 처리(`is_completed = true`)는 어떤 상태에서도 수행할 수 있다.
- 완료 처리 시 `completed_at` 기준으로 COMPLETED_ON_TIME 또는 COMPLETED_LATE가 결정된다.
- **OVERDUE 상태에서의 완료 처리는 반드시 COMPLETED_LATE로만 귀결된다.** (이미 `TODAY > end_date`이므로 `completed_at <= end_date` 성립 불가)
- 완료 해제(`is_completed = false`)는 언제든 가능하며, 해제 시 원래의 기간 기반 상태(UPCOMING / IN_PROGRESS / OVERDUE)로 복귀한다.

**통계 지표 정의 (UC-Q-03)**

| 지표명 | 산출 공식 |
|--------|----------|
| 전체 건수 | 사용자의 전체 Todo 수 |

| 기간 내 완료율 | `COMPLETED_ON_TIME / (COMPLETED_ON_TIME + COMPLETED_LATE) × 100` (%) — 완료된 항목 중 기간 내 완료 비율 |
| 기간 초과 건수 | OVERDUE + COMPLETED_LATE 건수 |

## 6. 비기능 요구사항 (NFR)

### 성능

| NFR ID | 요구사항 |
|--------|---------|
| NFR-P-01 | 할일 목록 조회 API 응답 시간은 P95 기준 500ms 이내여야 한다 |
| NFR-P-02 | 할일 등록/수정/삭제 API 응답 시간은 P95 기준 300ms 이내여야 한다 |

### 보안

| NFR ID | 요구사항 |
|--------|---------|
| NFR-S-01 | 비밀번호는 bcrypt 알고리즘으로 해시 처리하여 저장한다. 평문 저장 금지 |
| NFR-S-02 | JWT 액세스 토큰 유효 기간은 30분, Refresh Token 유효 기간은 7일로 한다 |
| NFR-S-03 | 모든 API 엔드포인트(인증 제외)는 유효한 액세스 토큰 없이 접근 시 401을 반환한다 |
| NFR-S-04 | HTTPS 통신을 필수로 한다. HTTP 요청은 HTTPS로 리다이렉트한다 |

### 데이터 보존

| NFR ID | 요구사항 |
|--------|---------|
| NFR-D-01 | 사용자가 계정을 삭제하면 해당 사용자의 모든 할일 데이터를 즉시 삭제한다 (CASCADE DELETE) |
| NFR-D-02 | 삭제된 데이터는 복구되지 않으므로 삭제 전 확인 절차를 UI에서 제공해야 한다 |

## 부록: 용어 정의 (유비쿼터스 언어)

| 용어 | 정의 |
|------|------|
| 할일 (Todo) | 사용자가 수행하고자 하는 개별 작업 단위. 반드시 시작일과 종료일을 가진다. |
| 시작일 (start_date) | 할일을 시작하는 예정 날짜. |
| 종료일 (end_date) | 할일을 완료해야 하는 마감 날짜. |
| 완료 (Completion) | 사용자가 할일을 수행 완료했다고 명시적으로 선언한 행위. |
| 기간 내 완료 | 종료일 이전 또는 당일에 완료 처리된 할일 (COMPLETED_ON_TIME). |
| 기간 초과 완료 | 종료일이 지난 후에 완료 처리된 할일 (COMPLETED_LATE). |
| 종료됨 (Overdue) | 종료일이 지났음에도 완료 처리되지 않은 할일. |
| 완료율 | 전체 할일 중 완료(기간 내 + 기간 초과) 처리된 할일의 비율. |
| 기간 내 완료율 | 완료된 할일 중 종료일 이내에 완료된 할일의 비율. |
| 통계 | 사용자의 전체 할일에 대한 상태별 건수, 완료율 등의 집계 정보. |
