**결과: ✅ PASS (동작 차이 있음)**

| 단계 | 기대 동작 | 실제 동작 | 판정 |
|------|----------|----------|------|
| 비밀번호 정책 실시간 검증 | 미충족 조건 표시, 버튼 비활성화 | ✓/○ 아이콘으로 실시간 표시, disabled | ✅ |
| 회원가입 버튼 클릭 | 계정 생성 후 안내 메시지 | "회원가입 성공 / 이메일 인증을 완료한 후 로그인해주세요." | ✅ |
| 이메일 인증 메시지 | "인증 이메일을 발송했습니다…" | 안내 문구 표시됨 | ✅ |
| 이메일 인증 완료 후 로그인 | 인증 완료 페이지 → 로그인 화면 이동 | **DB에 email_verify 컬럼 없음 — 인증 없이 로그인 가능** | ⚠️ |

- 회원가입 성공 안내 문구가 시나리오와 다름: "인증 이메일을 발송했습니다" 아닌 "이메일 인증을 완료한 후 로그인해주세요."

### US-11 — Silent Refresh (토큰 자동 갱신)

**결과: ⬜ SKIP**

액세스 토큰 유효기간(30분)이 만료되어야 테스트 가능한 시나리오로, 자동화 테스트 환경에서 즉시 검증 불가.

**수동 검증 방법:**
1. 로그인 후 30분 대기
2. 또는 브라우저 개발자 도구에서 localStorage의 accessToken을 만료된 값으로 교체
3. API 호출 시 자동으로 `/auth/refresh` 요청 후 원래 요청 재시도하는지 Network 탭에서 확인

## 동작 확인된 기능

- 회원가입 폼 및 비밀번호 정책 실시간 검증
- 로그인 오류 메시지 표시
- 할일 등록 (제목·설명·날짜)
- 할일 완료 처리 및 완료 해제 버튼 전환
- 확인 다이얼로그 (완료 처리, 삭제)
- 할일 수정 (설명, 종료일)
- 상태 필터 탭 (URL 쿼리 파라미터 연동)
- 정렬 기준/방향 드롭다운 (URL 쿼리 파라미터 연동)
- 통계 지표 표시 (완료율, 기간 내 완료율, 기간 초과 건수)
- 통계 → 필터 목록 연동 이동
- 목록에서 체크박스 직접 완료 처리
- 다국어 지원 (한국어/English/日本語/中文 셀렉터)
- 다크모드 전환 버튼
