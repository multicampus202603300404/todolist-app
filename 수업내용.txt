# 수업 내용 정리 — todolist-app 전체 개발 과정

날짜: 2026-04-02
프로젝트: todolist-app (개인별 할일 관리 애플리케이션)

---

## 1. 수업 주제

Claude Code의 서브에이전트를 활용하여 todolist-app의 설계 → 문서화 → 백엔드 구현 → 프론트엔드 구현까지
전체 소프트웨어 개발 라이프사이클을 실습

---

## 2. 진행 흐름 상세

#### STEP 1 — 도메인 정의서 (v1.0.0 → v1.1.0)

> **[프롬프트]** 너는 최고의 비즈니스 도메인 전문가야. "todolist-app"라는 이름의 개인별 할일관리 애플리케이션을 하려고 해. 다음 내용을 바탕으로 적절한 서브에이전트를 활용해 도메인 정의서 초안을 작성해줘.

- business-analyst 서브에이전트를 호출하여 도메인 정의서 초안(v1.0.0) 자동 작성
- 문서 구성: 프로젝트 개요, 도메인 모델(User/Todo), 비즈니스 규칙(BR-U/BR-T), 할일 상태 정의(5종), 유스케이스(13개), 유비쿼터스 언어

> **[프롬프트]** 전문 서브에이전트를 이용해 다음 5가지 항목을 기준으로 방금 작성한 도메인 정의서를 평가해줘 - 완전성, 명확성, 추적성, 검증 가능성, 유지보수성

- 5가지 기준으로 평가 수행 → 평균 **8.4/10**
- 주요 발견: 비기능 요구사항 부재, 예외 흐름 미정의, OVERDUE 상태 전환 논리 모순

> **[프롬프트]** 우선 개선 권고 사항을 적용하여 도메인 정의서를 재작성해줘

- 개선 3가지 적용: 예외 흐름 추가, NFR 섹션 신설, 논리 모순 수정
- 재평가 → 평균 **9.2/10** (+0.8점 향상)
- 산출물: `docs/1-domain-definition.md`

### Phase 5: 프론트엔드 구현 (FE-01 ~ FE-23)

- strict 모드 활성화, `npm run build` 성공 확인

> **[프롬프트]** `/frontend-resolver FE-02 부터 FE-23 까지 모두 진행해줘`

- 5단계 Phase로 서브에이전트 병렬 구현:

**Phase 1 (FE-02~04, FE-07): 기반 인프라**
- frontend-developer 서브에이전트 1개로 일괄 구현
- 패키지 설치 (zustand, @tanstack/react-query, axios, react-router-dom)
- 디렉토리 구조 생성 (18개 디렉토리 + 배럴 파일)
- CSS 변수/리셋/글로벌 스타일 (스타일 가이드 기반)
- TypeScript 타입 정의 (Todo, Auth, API, Statistics)
- Zustand 스토어 (authStore, toastStore)

**Phase 2 (FE-05~06, FE-08~09): API + Router**
- frontend-developer 서브에이전트 1개로 일괄 구현
- axios 인스턴스 + Silent Refresh 인터셉터 (401 큐잉 처리)
- API 함수 모듈 (authApi 4개 + todosApi 8개)
- QueryClient (staleTime 5분, 401 retry 비활성화)
- React Router + ProtectedRoute/PublicOnlyRoute
- 7개 페이지 placeholder 생성

**Phase 3 (FE-10~14): 공용 컴포넌트**
- frontend-developer 서브에이전트 1개로 5개 컴포넌트 일괄 구현
- Button (4 variant × 3 size + loading), Input/Textarea (forwardRef + 에러 상태)
- Dialog/ConfirmDialog (Portal + ESC + ARIA), Toast (Zustand 전역 호출 + 3초 auto-dismiss)
- LoadingSpinner (prefers-reduced-motion), EmptyState (icon/title/description/action)

**Phase 4 (FE-15~16): 인증 화면**

## 7. 핵심 학습 포인트

### 1. 에이전트 기반 개발 (Agent-Driven Development)
- 전문 서브에이전트(business-analyst, backend-developer, frontend-developer 등)에 역할을 분담하여 **병렬 작업**을 수행
- 한 번의 프롬프트로 여러 에이전트를 동시에 호출하여 BE/FE/DB를 병렬 구현
- 에이전트가 코드를 직접 생성하고, 테스트를 작성하고, 빌드를 검증하는 **자율적 개발 사이클** 수행

### 2. 문서 우선 개발 (Document-First Development)
- 코드 작성 전에 **9종의 설계 문서**를 먼저 완성
- 도메인 정의서 → PRD → 시나리오 → 구조 설계 → 아키텍처 → 실행계획 → 와이어프레임 → API 스펙 → 스타일 가이드
- 문서 품질을 **작성→평가→개선→재평가** 사이클로 반복 향상 (8.4/10 → 9.2/10)
- 구현 단계에서 문서를 참조하여 **일관된 코드 품질** 확보

### 3. 교차 검토 (Cross-Validation)
- 문서 간 **일관성 검증**이 구현 단계의 오류를 사전 방지
- PRD ↔ 시나리오 ↔ swagger ↔ 코드 4방향 검토로 **12건 이상의 불일치** 발견 및 수정
- 예시: ERD의 int 타입 오류, 페이지네이션 필드명 불일치, 완료 처리 화면 이동 방향 충돌

### 4. 실행계획 기반 추적 (Plan-Driven Tracking)
- 45개 Task를 **의존성 그래프**로 정렬하여 병렬 실행 순서 결정
- 268개 **체크박스**로 진행률을 가시화하고 완료 기준을 명확히 정의
- 각 Task에 **산출물, 완료 조건, 선행 의존성**을 명시하여 자기 검증 가능

### 5. 사용자 정의 Command (Custom Slash Commands)
- `/backend-resolver`, `/frontend-resolver`로 반복 작업 워크플로우를 **표준화**
- 문서 분석 → 코드 분석 → 계획 수립 → 테스트 작성 → 구현 → 테스트 수행 → 체크박스 업데이트
- 한 번의 프롬프트로 **8개 태스크를 일괄 처리** (예: BE-05~12, FE-02~23)

### 6. MCP 활용 (Model Context Protocol)
- `postgresql-mcp`로 DB를 **Claude Code 내에서 직접 조작** (테이블 생성, 데이터 조회, 연결 테스트)
- 마이그레이션 실행 결과를 즉시 검증 (컬럼 타입, 인덱스, 트리거 동작 확인)
- 시드 데이터 삽입 후 데이터 무결성 확인까지 **하나의 대화 흐름**에서 완료
